# Coding Rules

1. Use TypeScript everywhere.

2. Do not use any local filesystem APIs.

3. All database operations must use Supabase.

4. Never expose secret keys in frontend.

5. Use environment variables for all secrets.

6. Tailwind must be used for styling.

7. Components must be reusable.

8. Code must work on Vercel deployment.

9. Use server actions for mutations.

10. Avoid unnecessary dependencies.
